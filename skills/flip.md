# /flip — The Inversion Engine

You find the move that works BECAUSE it's the opposite of what everyone does. Not contrarianism for its own sake. Structural inversions where reversing the assumption produces a genuinely better outcome.

## The Canonical Example

**Established approach:** Minimize product returns. Returns = cost = bad.
**The flip:** Zappos made returns free, unlimited, both ways. Returns became trust. Trust became lifetime value. The company that embraced returns the most became the most valuable shoe retailer.

**Why the flip worked:** Returns weren't actually a cost. They were a trust barrier disguised as a cost. Removing the barrier increased purchase confidence so much that net revenue went UP despite higher return volume. The assumption ("returns are bad") was load-bearing but wrong.

## Preparation

Before inverting, you MUST deeply understand the current approach. Read relevant context. Understand WHY things are done this way. The best flips come from understanding the original logic so well that you can identify which part of it is secretly wrong.

## STAGE 1: THE ORTHODOXY

State the established approach with maximum specificity. Not a strawman. The BEST version of the conventional argument.

- **What everyone does:** [Specific description]
- **Why they do it:** [The reasoning. Make it sound smart, because it usually is.]
- **The load-bearing assumption:** [The single belief that, if wrong, collapses the whole approach]
- **Who profits from this assumption:** [Sometimes the assumption persists because someone benefits from it being unquestioned]

## STAGE 2: SEVEN INVERSIONS

Apply each inversion type. Not all will produce viable flips. Mark duds as "NO FLIP" and move on. But genuinely attempt each one.

### 1. SEQUENCE FLIP
What if you did the last step first? What if the output came before the input? What if the result preceded the effort?
Ask: "What if the order was completely reversed?"

### 2. ROLE SWAP
What if the buyer was the seller? What if the student taught the teacher? What if the product was the customer?
Ask: "What if the roles were reversed?"

### 3. COST-AS-FEATURE
What if the thing everyone tries to minimize was actually the value? What if the "problem" was the product?
Ask: "What if the biggest cost was actually the biggest asset?"

### 4. CONSTRAINT REMOVAL
What if the thing everyone accepts as a hard constraint was actually optional? What if you just... didn't do it?
Ask: "What if this 'requirement' simply didn't exist?"

### 5. AUDIENCE INVERSION
What if you built for the people everyone else ignores? What if the "wrong" customer was actually the right one?
Ask: "What if you targeted the exact opposite audience?"

### 6. METRIC FLIP
What if you optimized for the opposite metric? What if you maximized what everyone minimizes?
Ask: "What if success meant the opposite of what everyone measures?"

### 7. ADDITION BY SUBTRACTION
What if removing a feature, step, or option made the product more valuable? What if less was genuinely more?
Ask: "What if you deleted the thing everyone thinks is essential?"

### IMPLEMENTATION

### WHAT KEEPS THE FLIP FROM BEING STUPID
[The line between a brilliant inversion and a dumb contrarian take is thin. What makes this one land on the right side? What would push it to the wrong side?]

### THE LITMUS TEST
- Does the flip produce a better outcome, not just a different one?
- Is the inverted assumption genuinely wrong, or just unpopular?
- Would a smart person's first reaction be "that's backwards" followed by "...wait, actually"?
- Does it compound? (Does the inversion get stronger over time?)

## STAGE 4: THE ANTI-FLIP (Mandatory)

For the strongest flip, argue against it. Why might the original approach actually be right? What conditions would make the flip fail? State the strongest case for NOT flipping. If this case is more compelling than the flip itself, say so honestly.

## Rules

- No preamble. Start with Stage 1 immediately.
- ONE strongest flip, not a list. Convergent, not divergent.
- Every flip must be SPECIFIC. "Do the opposite" is not a flip. "Charge customers for returning products instead of making returns free" is a flip (even if it's a bad one).
- The line between insight and contrarianism is whether the flip produces a BETTER outcome. Not different. Better. Hold yourself to this.
- If no inversion produces a genuinely better outcome, say so. "The current approach is actually right" is a valid conclusion. Not every orthodoxy deserves to be flipped.
- Label the load-bearing assumption explicitly. If you can't identify it, the flip will be shallow.
